# คู่มือการตั้งค่า Database (Database Setup Guide)

โปรเจกต์นี้ใช้งาน **PostgreSQL** เป็นฐานข้อมูลหลัก และใช้ **Prisma ORM** ในการจัดการ Schema และ Query ต่างๆ

ด้านล่างนี้คือขั้นตอนการตั้งค่าเพื่อรันฐานข้อมูลบนเครื่องตัวเอง (Local) แบบละเอียดครับ

---

## สิ่งที่ต้องมี (Prerequisites)

1. ติดตั้ง [PostgreSQL](https://www.postgresql.org/download/) ในเครื่องให้เรียบร้อย (ในโปรเจกต์นี้ทดสอบบนเวอร์ชัน 18 แต่เวอร์ชัน 14+ ก็สามารถใช้งานได้)
2. จำ Username และ Password ของ PostgreSQL ที่ตั้งไว้ตอนติดตั้ง (ค่าเริ่มต้นมักจะเป็น username: `postgres`)
3. ติดตั้ง Node.js และรัน `npm install` ในโปรเจกต์ให้เรียบร้อย

---

## ขั้นตอนที่ 1: สร้างฐานข้อมูล (Create Database)

คุณต้องสร้างฐานข้อมูลเปล่าๆ ขึ้นมาก่อน สามารถทำได้ 2 วิธี:

**วิธีที่ 1: ใช้ pgAdmin (แบบมีหน้า UI)**

1. เปิดโปรแกรม pgAdmin ที่แถมมาตอนติดตั้ง PostgreSQL
2. เข้าสู่ระบบด้วย Password ของคุณ
3. คลิกขวาที่ `Databases` -> `Create` -> `Database...`
4. ตั้งชื่อฐานข้อมูลว่า `pirahus` (หรือชื่ออื่นตามต้องการ) แล้วกด Save

**วิธีที่ 2: ใช้ Command Line (psql)**
เปิด Terminal หรือ Command Prompt แล้วพิมพ์คำสั่งด้านล่าง (ต้องมีการเซ็ต Path ของ psql ไว้แล้ว):

```bash
psql -U postgres -c "CREATE DATABASE pirahus;"
```

_(ระบบจะถามรหัสผ่าน ให้พิมพ์รหัสผ่านของคุณลงไป)_

---

## ขั้นตอนที่ 2: ตั้งค่าไฟล์ .env

ไปที่โฟลเดอร์ root ของโปรเจกต์ ค้นหาไฟล์ `.env` (หากไม่มีให้ก็อปปี้จาก `.env.example` มาสร้างเป็น `.env`)

แก้ไขตัวแปร `DATABASE_URL` ให้ชี้ไปที่ Local Database ของคุณ โดยใช้รูปแบบ:
`postgresql://<USERNAME>:<PASSWORD>@localhost:5432/<DATABASE_NAME>`

**ตัวอย่าง:**

```env
DATABASE_URL="postgresql://postgres:asnawee2550@localhost:5432/pirahus"
```

---

## ขั้นตอนที่ 3: นำ Schema ไปสร้างตารางใน Database

เมื่อเชื่อมต่อสำเร็จแล้ว เราจะใช้ Prisma เพื่อสร้างโครงสร้างตารางต่างๆ รันคำสั่งนี้ใน Terminal:

```bash
npx prisma db push
```

_คำสั่งนี้จะทำการเปรียบเทียบไฟล์ `prisma/schema.prisma` กับ Database ปัจจุบัน แล้วสร้างหรือปรับปรุงตารางให้ตรงกัน_

จากนั้นให้รันคำสั่ง Generate เพื่ออัปเดต Prisma Client ให้โปรเจกต์เรารู้จักโครงสร้างล่าสุด:

```bash
npx prisma generate
```

---

## ขั้นตอนที่ 4: เพิ่มข้อมูลเริ่มต้น (Seeding Data)

เนื่องจากข้อมูลเริ่มต้นมีความสำคัญ เราจึงไม่ได้เอาไฟล์ข้อมูล (`seed-data.json`) ขึ้น Git ไปด้วย 
**หากคุณได้รับไฟล์ `seed-data.json` มาจากเพื่อนร่วมทีม ให้คุณนำไฟล์นั้นไปวางไว้ในโฟลเดอร์ `prisma` (ให้อยู่ที่ `prisma/seed-data.json`)**

จากนั้น คุณสามารถเอาข้อมูลในไฟล์นั้นใส่ลงใน Database ของคุณได้ง่ายๆ โดยการรันคำสั่ง:

```bash
npx prisma db seed
```

_คำสั่งนี้จะไปอ่านข้อมูลจากไฟล์ JSON และทำการสร้างข้อมูลทั้งหมดลงใน Local Database ของคุณอัตโนมัติ (หากมีข้อมูลไหนซ้ำ ระบบจะทำการอัปเดตให้แทน)_
